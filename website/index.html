Project Focus

Open Infrastructure for Real-World RL

Open real-world RL pipeline

Runnable workflows on SO101 and AgileX PiPER covering collection, offline value/policy training, and iterative deployment.

Reproducible assets

Codebase, model layout, dataset structure, and command-line templates are released to reduce setup friction and improve reproducibility.

Community co-evolution

Reproduce existing real-world RL methods, extend value/policy variants, and continuously publish experiments and benchmarks for collective progress.

Method

6-Stage Closed-Loop Pipeline

  1. 01

    System preparation to ensure robot, sensors, and control stack are ready.

  2. 02

    Interactive data collection with human guidance and episode-level outcomes.

  3. 03

    Value modeling to estimate trajectory quality from accumulated data.

  4. 04

    Signal annotation to convert value outputs into usable training supervision.

  5. 05

    Policy update and deployment to run the improved policy in real tasks.

  6. 06

    Data refresh and iteration to continue improving robustness and success rate.

Architecture

LeRobot-Aligned VLA + Value Loop

Evo-RL reuses LeRobot's practical data and inference stack while adding value-guided filtering signals (`value`, `advantage`, `acp_indicator`) to improve iterative policy training.

The same design supports single-GPU and accelerate-based multi-GPU workflows for both value and policy stages.

Reproducibility

Core CLI Commands

Human-in-loop Record

lerobot-human-inloop-record \
  --robot.type=<ROBOT_TYPE> \
  --teleop.type=<TELEOP_TYPE> \
  --dataset.repo_id=<HF_USER_OR_ORG/DATASET> \
  --dataset.single_task="<TASK_TEXT>" \
  --dataset.num_episodes=<NUM_EPISODES> \
  --display_data=true

Value Training

lerobot-value-train \
  --dataset.repo_id=<HF_USER_OR_ORG/DATASET> \
  --value.type=<VALUE_TYPE> \
  --batch_size=<BATCH_SIZE> \
  --output_dir=outputs/value_train/<RUN_NAME> \
  --job_name=<RUN_NAME>

Policy Training

lerobot-train \
  --dataset.repo_id=<HF_USER_OR_ORG/DATASET> \
  --policy.type=<POLICY_TYPE> \
  --steps=<TRAIN_STEPS> \
  --acp.enable=true \
  --acp.indicator_dropout_prob=0.3 \
  --output_dir=outputs/train/<RUN_NAME>
